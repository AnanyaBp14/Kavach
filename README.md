# Kavach Shield

A working prototype of the flagship module from the **Kavach — Digital Public Safety Intelligence Grid**
concept: a real-time AI risk classifier for India's "digital arrest" scam calls, submitted for the
*AI for Digital Public Safety* challenge.

This is a **full-stack** version — a small Express backend holds the Groq API key server-side and
exposes one endpoint; the frontend (plain HTML/CSS/JS, no build step) calls that endpoint instead of
talking to any AI provider directly. It runs on **Groq's free tier** (no credit card required), so you
can get it running end-to-end at zero cost.

```
kavach-project/
├── server/
│   ├── server.js          # Express app: serves the frontend + POST /api/assess
│   ├── package.json
```

## Quick start

```bash
cd server
npm install
cp .env.example .env
# edit .env and paste your Groq API key
npm start
```

Then open **http://localhost:3000** — that's the whole app, frontend and backend on one port.

### Getting a free Groq API key

1. Go to **console.groq.com** and sign up (email or Google/GitHub — no credit card).
2. Open **console.groq.com/keys** and create a new key.
3. Paste it into `server/.env` as `GROQ_API_KEY=gsk_...`.

Groq's free tier runs on rate limits, not a spend cap or expiring credits — it doesn't run out the way a
trial credit does. The default model here, `llama-3.3-70b-versatile`, is on the free tier. The catch is a
fairly tight per-minute cap (tens of requests per minute, shared across your whole app), which is why the
server has a conservative built-in rate limiter — see **Configuration** below if you need to raise it
after upgrading to Groq's paid Developer tier.

## How it works

1. The browser sends `{ story, signals, lang }` to `POST /api/assess`.
2. The server builds a system prompt calibrated against known MHA/NCRB digital-arrest scam patterns,
   calls Groq (`llama-3.3-70b-versatile` by default) via the official `groq-sdk`, using JSON mode so the
   model is constrained to return structured output, and validates the JSON it gets back.
3. The server attaches a simulated intelligence-alert payload (timestamp, evidence hash, routing —
   illustrating what a production integration with MHA / NCRB / telecom systems would look like) and
   returns everything to the browser.
4. The frontend renders the risk gauge, verdict, indicators, and recommended actions, and always keeps
   the **1930** helpline and `cybercrime.gov.in` visible.

The API key never reaches the browser — this is the difference between this version and a single-file
demo that calls an AI API directly from client-side JavaScript, which isn't something you can deploy
without leaking the key.

## Configuration

| Variable | Required | Default | Notes |
|---|---|---|---|
| `GROQ_API_KEY` | Yes | — | Free, no card, from [console.groq.com/keys](https://console.groq.com/keys) |
| `GROQ_MODEL` | No | `llama-3.3-70b-versatile` | Any Groq-hosted chat model that supports JSON mode |
| `PORT` | No | `3000` | |

The in-app rate limiter (`RATE_LIMIT_MAX` in `server.js`) defaults to 10 requests/minute per visitor to
stay comfortably under Groq's shared free-tier cap. If you upgrade to Groq's paid Developer tier (adds a
card, unlocks ~10x the rate limit + a token discount), you can raise this.

## Notes on taking this further

- **Rate limiting** is in-memory per IP for demo purposes — swap in Redis or a proper API gateway before
  any real traffic.
- **Persistence**: nothing is stored right now. A production version would log verdicts (with consent)
  for the model-audit trail and the classifier feedback loop described in the architecture diagram.
- **Auth**: there is none — anyone who can reach the server can call `/api/assess`. Fine for a local demo
  or an internal pilot behind a VPN; add auth before any public deployment.
- **Model swap**: this is provider-agnostic by design — `server.js` isolates all the AI-calling logic in
  one place, so swapping Groq for another OpenAI-compatible provider (or back to a provider like
  Anthropic) is a change to the imports and the `chat.completions.create` / equivalent call, not a
  frontend change.
- **Multi-agent modules** (Counterfeit Vision, Graph Intelligence, Geospatial, Voice Spoof) referenced in
  `docs/kavach-architecture.svg` are not implemented here — this repo is the Phase 1 prototype only.

## Disclaimer

This is a hackathon prototype. Risk assessments are generated by an AI model and are advisory only, not
a substitute for official verification. No genuine Indian government agency (CBI, ED, Police, Customs,
RBI) conducts arrests or demands money over a video call.
